# 주휴수당 관련 로직 정리

이 문서는 저장소 내 **주휴수당(weekly allowance)** 과 연결된 코드·설정·UI를 파일별로 묶어 정리한 것이다.  
(작성 기준: `rule_engine.py`, `payroll_calculator.py`, `app.py`, `templates/result.html`, `google_sheet_exporter.py`, `send_payroll_emails.py`, `policies.yaml`, `흐름도.md`)

---

## 1. 데이터 흐름 요약

```mermaid
flowchart LR
  DS[daily_summary.csv]
  RE[rule_engine.py]
  WA[weekly_allowance_result.csv]
  PC[payroll_calculator.py]
  PR[payroll_result.csv]
  DS --> RE
  RE --> WA
  DS --> PC
  WA --> PC
  PC --> PR
```

- **산정(분 단위)**: `rule_engine.build_weekly_allowance()` → `weekly_allowance_minutes`를 주(월요일 시작)·직원별로 계산해 `weekly_allowance_result.csv`에 저장한다.
- **급여 표·금액**: `payroll_calculator.main()`이 위 결과를 읽어 `주휴1`, `주휴2`, … 컬럼(시간)과 `weekly_allowance_pay`, `weekly_allowance_hours`, `total_pay`를 만든다.
- **웹 편집·재계산**: `app.py`의 `_recalc_pay_from_date_columns`와 `templates/result.html`의 `recalcRow()`가 **일자 셀 + 주휴N 컬럼**으로 합계를 맞춘다.

## 2. 핵심 산정 규칙 — `rule_engine.py`

| 항목 | 내용 |
|------|------|
| 파일 | `rule_engine.py` |
| 진입점 | `main()` → `build_weekly_allowance(daily)` |
| 출력 | `output/weekly_allowance_result.csv` (`employee_id`, `week_start`, `total_work_minutes`, `weekly_allowance_minutes`) |
| 주 단위 | **월요일을 주 시작**으로 하고, 주휴일은 **일요일** 기준으로 급여 산정기간과 맞춘다(모듈 상단 주석·`payroll_calculator`와 연동). |

### 2.1 상수

- `MAX_DAILY_MINUTES` = 480 — 일별 반영 상한(8시간).
- `MAX_WEEKLY_ALLOWANCE_MINUTES` = 480 — 주휴 산정 시간 상한(8시간).
- `MIN_WEEKLY_MINUTES_FREELANCER` = 900 — 프리랜스 주간 **15시간 미만**이면 해당 주 `weekly_allowance_minutes = 0`.

### 2.2 공통 공식(비프리랜스 또는 프리랜스 공휴일 제외 후)

1. 해당 주의 **월~금 행만** 사용(`weekdays_df`).
2. 일별 `net_minutes`를 **480분으로 캡**한 뒤 합산 → `capped_total`.
3. `allow_min = min(round(capped_total / 5), MAX_WEEKLY_ALLOWANCE_MINUTES)`.

### 2.3 프리랜스 전용 (`_is_freelancer`)

- **평일 한국 공휴일**(`leave_merger.get_weekday_public_holidays_kr`)에 해당하는 날은 주휴 산정 합계에서 **제외**(그날 근무는 일별에서 야근 등으로 이미 처리된다는 전제, 모듈 주석 참고).
- 주간 `capped_total`이 **15시간(900분) 미만**이면 `allow_min = 0`.

### 2.4 무급 오프(`day_highlight == "unpaid_leave"` & `net_minutes == 0`)

- 같은 주에 이런 날이 **2일 이상**이면 해당 주 `allow_min = 0`.
- 급여 산정기간 안에서 **0시간 무급 오프가 처음 생긴 주**만 주휴를 인정하고, **그 이후 주**에 다시 0시간 무급 오프가 있으면 해당 주 `allow_min = 0` (`first_zero_off_week_by_emp`).

### 2.5 의존성

- `leave_merger.get_weekday_public_holidays_kr` (프리랜스 공휴일 제외용).
- `attendance_normalizer.load_contract_config` (프리랜스 판별용 `_employee_contracts_cache`).

## 3. 급여 표·CSV — `payroll_calculator.py`

| 항목 | 내용 |
|------|------|
| 입력 | `daily_summary.csv`, `weekly_allowance_result.csv` |
| 출력 | `payroll_result.csv` |

### 3.1 산정기간·주휴 컬럼 노출

- `_infer_payroll_period(daily)`로 **전월 25일 ~ 당월 24일** 형태의 산정기간을 잡는다.
- **해당 주의 일요일이 산정기간 안에 있을 때만** 그 주의 주휴 금액을 이번 급여에 합산(`weeks_in_period`, `_week_in_period`).
- 날짜 컬럼 헤더: 산정기간 **이전** 평일은 `1/20\n주휴용`처럼 **두 줄 헤더** — 참고용이며 **기본급·야근에는 넣지 않는다** (`header_for_date`).

### 3.2 직원별 집계

- 산정기간 안의 일자만 `base_pay` / `overtime_pay` 합산(`calc_daily`, 프리랜스 공휴일은 야근 전부 등).
- `weekly_allowance_minutes`는 산정기간에 들어오는 주만 필터해 **합산** → `allow_pay = round(round(분/60,1) * HOURLY_WAGE, 0)` (`HOURLY_WAGE` = 11000).
- `주휴N` 컬럼: 해당 주의 `weekly_allowance_minutes / 60`을 소수 1자리로 표시.

### 3.3 컬럼명

- `weekly_allowance_pay`, `weekly_allowance_hours`, `total_pay` = 기본 + 야근 + 주휴.

## 4. 웹 앱 재계산 — `app.py`

| 함수/영역 | 역할 |
|-----------|------|
| `_is_non_half_hour` | 주휴 **제외** 일자 컬럼에서 `.0`/`.5`가 아닌 시간 → UI 빗금 표시용. |
| `_recalc_pay_from_date_columns` | 편집된 표에서 **일자 컬럼**만으로 기본급·야근 재산정. **`주휴`로 시작하는 컬럼·`주휴용` 포함 컬럼은 일자 합산에서 제외**. |
| 주휴 컬럼 합산 | `주휴1`, `주휴2`, … (이름에 `주휴용` 없음) 값을 **시간 단위로 합산** → `weekly_allowance_hours`, `weekly_allowance_pay = 합계시간 × 11000`, `total_pay` 갱신. |
| 구글 시트보내기 등 | 동일 로직으로 `holiday_dates`·`date_col_to_date`와 함께 재계산하는 경로가 있음(프리랜서 평일 공휴일 야근 처리). |

관련 `pay_cols`: `weekly_allowance_pay`, `weekly_allowance_hours`.

## 5. 결과 화면 UI — `templates/result.html`

| 영역 | 내용 |
|------|------|
| CSS | `주휴용` 라벨 폰트(`th-weekly-label`), 주휴 컬럼 배경(`col-weekly`), 일자 컬럼 비0.5단위 빗금(`text-non-half`). |
| 테이블 | `payroll_cell_list`에서 `typ == 'weekly'` → 주휴 컬럼, `주휴용` in 헤더 → `data-is-prior-weekly` 등으로 **기본급 합산 제외** 표시. |
| `recalcRow()` | 서버 `_recalc_pay_from_date_columns`와 동일 상수(시급 11000, OT 1.5배, 8h cap). `주휴용`이 들어간 `data-col`은 일자 합산에서 스킵. `data-is-weekly="true"` 셀 합계로 주휴수당·산정시간·총액 실시간 반영. |

| `build_email_sheet_data` | 컬럼 `텍스트_주휴수당`; **정규직**은 `fill_text`가 꺼져 비움, 상용/프리랜서는 채움. |

## 8. 선행 데이터(간접 연관)

주휴 산정의 입력은 `daily_summary.csv`의 **`net_minutes`**, **`day_highlight`** 이다.

| 파일 | 연관 내용 |
|------|-----------|
| `leave_merger.py` | 유급/무급/평일 공휴일 처리 → `net_minutes`, `day_highlight`가 주휴 규칙(무급 0h, 프리랜스 공휴일)에 영향. |
| `attendance_normalizer.py` | 근무·휴게 집계 및 계약별 `scheduled_minutes` (휴가 전용 행 등과 연계). |

## 9. `policies.yaml` / `흐름도.md` 참고

| 파일 | 비고 |
|------|------|
| `policies.yaml` | `weekly_allowance_policy`, `weekly_allowance_standard` 등 **계약 타입별 정책 형태로 정의되어 있으나**, 현재 **Python 파이프라인 어디에서도 import/사용되지 않음** (`policies.yaml` 문자열 grep 기준). 실제 주휴 규칙은 **`rule_engine.py`에 하드코딩**되어 있다. |
